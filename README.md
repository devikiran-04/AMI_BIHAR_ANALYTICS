# ⚡ AMI BIHAR ANALYTICS

A high-performance analytics dashboard and ETL orchestration toolkit designed for the Bihar Advanced Metering Infrastructure (AMI) project. This system processes and visualizes millions of smart meter telemetry events daily, providing critical insights into meter data management, billing reconciliation, and vendor performance.

## 🎯 Project Overview

This repository contains the analytics frontend and the underlying SQL logic for monitoring one of India's largest smart metering rollouts. It is engineered to handle the massive data volume generated by over **1.6+ million smart prepaid meters** across Bihar .

The project addresses two core functions:
1.  **ETL Processing:** High-performance data pipelines (using Pentaho Data Integration) that transform raw meter events into structured relational data with minimal latency.
2.  **Analytics & Reporting:** A web-based interface providing real-time operational visibility into data quality, vendor performance, and billing success rates.

## 🛠️ Technology Stack

| Category | Technology | Purpose |
| :--- | :--- | :--- |
| **Frontend** | Streamlit | Interactive web interface for reports and data exploration |
| **Backend Logic** | Python 3.x, SQLAlchemy, Pandas | Database connectivity and data manipulation |
| **Data Processing** | PostgreSQL / TimescaleDB | High-performance time-series data storage for meter telemetry  |
| **ETL Orchestration** | Pentaho Data Integration (Kettle) | Complex data workflow automation and CDC-style data extraction |
| **Infrastructure** | AWS / On-Premise | Scalable cloud architecture for data ingestion |

## 🚀 Getting Started

### Prerequisites
- Python 3.8+
- Access to the PostgreSQL/TimescaleDB database instance
- Git

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/devikiran-04/AMI_BIHAR_ANALYTICS.git
    cd AMI_BIHAR_ANALYTICS
    ```

2.  **Set up a virtual environment**
    ```bash
    python -m venv venv
    source venv/bin/activate   # On Windows: venv\Scripts\activate
    ```

3.  **Install dependencies**
    ```bash
    pip install -r requirements.txt
    ```

### Configuration

Create a `config.env` file in the root directory with your database credentials:
```
DB_HOST=your_postgres_host
DB_NAME=nbpdcl_db
DB_PORT=5432
DB_USER=your_username
DB_PASSWORD=your_secure_password
```

### Running the Dashboard

Launch the Streamlit application:
```bash
streamlit run app.py
```
Access the dashboard at `http://localhost:8501`.

## 📊 Features & Reports

The sidebar provides access to the following key operational reports:

| Report Name | Description | Data Source |
| :--- | :--- | :--- |
| **MDMS Stage Counts** | Daily count of successfully processed Load Profile (LP) and Energy Data (ED) files. | `mdms` schema |
| **MDMS BLP/DLP Vendor Counts** | Vendor-wise breakdown of valid vs. invalid meter blocks for billing periods. | `mdms` schema |
| **HES Profile Counts** | Summary of distinct meters reporting ED, LP, Instant, and EOB data. | `fep` schema |
| **SPM Billing** | Reconciliation of prepaid meters, billed vs. unbilled counts, and stopped meters. | `spm` schema |
| **NFMS Reports** | Comparison of transaction meter counts between source systems and the NFMS layer. | `nfms` schema |
| **DB Size** | Real-time size of the `nbpdcl_db` database for capacity planning. | PostgreSQL System Catalog |
